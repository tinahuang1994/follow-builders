# Make Deck Prompt — 小白版 HTML Deck（Tina 版产出格式）

把 `prepare-digest.js` 输出的 JSON（`x` 推文 + `blogs` 官博 + `tencent` 腾讯条目）remix 成一份**中文小白版 HTML deck**，而不是文字 digest。

## 输出
- 一个 960×540 的横向 slide deck，HTML 单文件，存到用户配置的输出目录（默认 `AI日报-YYYY-MM-DD-小白版.html`）。
- 复用仓库里最近一份 `*-小白版.html` 的 `<style>` 与结构：cream/bronze 配色、Songti 标题、`card`(who+pt) + `sowhat` + `src` 脚注、底部 nav + dots、`#counter` 写「N / 总页数」。
- 结构：封面 + 5–8 条 + 收尾，约 10 页。

## 写法铁律（和原版一致 + Tina 的加法）
- **融成一个声音**：多源（X / 腾讯 / 官博）融合后当成「只有一个信息源」写——正文不提来源数量、不给单条打来源标签、不按来源切页。
- **页脚 `.src` 只引一手来源**：官方博客 / 论文 / 公告 / 当事人原始推文（用 JSON 里的 `url`）/ 原始播客。绝不把「腾讯 / Zara / 搜狐 / 聚合站」当来源。拿不到一手 `url` 的那条不收。
- **绝不编造**：只用 JSON 里真实存在的内容；没 `url` 不收；不臆测某人「在憋什么大招」。
- 人名用全名 + 角色（用推文 `bio` 推断身份，如「Box CEO Aaron Levie」），不只写姓、不写 @handle。

## ⭐ Tina 的独家价值：每条都要「So what · 对你」（个性化）
- 读用户 config 里的 `userContext`（安装时问的「你是做什么的 / 在建什么」）。
- 每条新闻配一个 `sowhat` 框，标题「So what · 对你」，把这条**落到该用户自己的处境**：他的工作、他在建的东西、他能怎么用。
- 若 `userContext` 为空：退化成通用解读「这对你意味着什么」，但仍要具体、可行动，不空泛。
- 这是这份 deck 区别于一般 AI 新闻的核心——务必每条都做，且讲到外行能懂。

## 选题
- 优先：用户领域相关的工具/模型动态、省钱提效打法、安全与纪律、鼓舞性的真实用例。
- 跳过：无实质的日常推文、纯转发、广告、「好棒的活动」类。

## 收尾页
- 用 3–4 个要点把当天信号串成一句话，再给一个总的 So-what。
